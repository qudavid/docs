# API Reference

This section provides detailed API endpoint documentation for all Predexon services.

## Base URLs

- **REST API**: `https://api.predexon.com/v1`
- **WebSocket**: `wss://stream.predexon.com/v1`

## Authentication

All API requests require authentication using Bearer tokens in the Authorization header:

```bash
Authorization: Bearer YOUR_API_KEY
```

Contact us to obtain an API key for accessing Predexon services.

## Data Formats

### Timestamps
All timestamps are in ISO-8601 UTC format:
```
2025-10-06T19:21:02Z
```

### Numbers
All numeric fields use string representation for fixed precision:
```json
{
  "price": "0.5234",
  "volume": "12345.67"
}
```

### Pagination
Cursor-based pagination is used throughout:
```bash
?cursor=<opaque_string>&limit=<integer>
```

Response includes `next_cursor` for fetching subsequent pages.

## Rate Limits

API rate limits are enforced per API key. Exceeding rate limits returns a `429 Too Many Requests` error.

## Error Handling

Standard HTTP status codes:
- `400` - Bad Request
- `401` - Unauthorized
- `403` - Forbidden
- `404` - Not Found
- `422` - Unprocessable Entity
- `429` - Too Many Requests
- `5xx` - Server Errors

See [Error Reference](/support/errors) for detailed error codes and troubleshooting.
