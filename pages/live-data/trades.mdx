# Recent Trades

> **GET `/v1/trades/{market_uid}`** — Retrieve paginated trade prints for a market with optional time filters for replay.

- **Base URL:** `https://api.predexon.com/v1`
- **Auth:** `Authorization: Bearer <token>`
- **Time:** All timestamps are **UTC ISO-8601** (e.g., `2025-10-06T19:21:02Z`)
- **Rate Limits:** 100 requests/minute per API key. Returns `429` with `Retry-After` header (seconds) when exceeded. Use exponential backoff on 429/503 responses.

## Endpoint

```http
GET /v1/trades/{market_uid}
```

## Path parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `market_uid` | string | Canonical market identifier. Required. |

## Query parameters

All parameters are optional.

| Parameter | Type | Description |
|-----------|------|-------------|
| `start` | string | Inclusive start timestamp filter (`ISO-8601`). |
| `end` | string | Exclusive end timestamp filter (`ISO-8601`). |
| `limit` | integer | Results per page (default 100, max 1000). |
| `cursor` | string | Opaque cursor returned in `next_cursor`. |

## Field reference

| Field | Type | Description |
|-------|------|-------------|
| `market_uid` | string | Canonical market identifier for the trade collection. |
| `venues` | object | Map of venue code → venue-specific trade results. |
| `venues.polymarket` | object \| null | Trade data from Polymarket (null if not listed). |
| `venues.kalshi` | object \| null | Trade data from Kalshi (null if not listed). |
| `venues[*].market_slug` | string | Polymarket market slug (when applicable). |
| `venues[*].market_ticker` | string | Kalshi market ticker (when applicable). |
| `venues[*].trades` | array | Trades for that venue (newest first). |
| `venues[*].trades[].ts` | string (ISO-8601) | Execution timestamp. |
| `venues[*].trades[].outcome` | string | Outcome traded (e.g., `YES`, `NO`). |
| `venues[*].trades[].price` | string | Execution price. |
| `venues[*].trades[].size` | string | Trade size. |
| `venues[*].trades[].venue_trade_id` | string | Venue-native trade identifier. |
| `venues[*].next_cursor` | string \| null | Cursor for next page on that venue (null when complete). |

<Info>
**Pagination tips**

- Use `start`/`end` for bounded historical pulls, then iterate with each venue's `next_cursor` for deep pagination.
- Store the newest processed `ts` and `venue_trade_id` per venue to avoid duplicates when polling.
- Combine with [Stream Quotes](/pages/live-data/stream-quotes) or strategy logic to track fills vs. mid-price moves.
</Info>

## Examples

<CodeGroup>

```bash cURL
curl -s "https://api.predexon.com/v1/trades/mkt_evt_crypto_2025_07_01_BTC_above_75000_yes?limit=200" \
  -H "Authorization: Bearer $PREDX_TOKEN"
```

```javascript JavaScript
const base = "https://api.predexon.com/v1";

async function listTrades(marketUid, params = {}) {
  const url = new URL(`${base}/trades/${marketUid}`);
  Object.entries(params).forEach(([key, value]) => url.searchParams.set(key, value));

  const res = await fetch(url, { headers: { Authorization: `Bearer ${process.env.PREDX_TOKEN}` } });
  if (!res.ok) throw new Error(`${res.status} ${await res.text()}`);
  return res.json();
}
```

```python Python
import os
import requests

def list_trades(market_uid: str, **params) -> dict:
  res = requests.get(
      f"https://api.predexon.com/v1/trades/{market_uid}",
      headers={"Authorization": f"Bearer {os.environ['PREDX_TOKEN']}"},
      params=params,
      timeout=30,
  )
  res.raise_for_status()
  return res.json()

print(list_trades("mkt_evt_crypto_2025_07_01_BTC_above_75000_yes", limit=200))
```

</CodeGroup>

Each venue returns its own `trades` array and cursor. Venues where the market is unavailable are returned as `null`.

## Errors

All errors return standard HTTP status codes with JSON error responses:

```json
{
  "error": {
    "type": "invalid_parameter",
    "message": "limit must be <= 1000",
    "param": "limit"
  }
}
```

| HTTP Status | Error Type | Description | Common Causes |
|-------------|------------|-------------|---------------|
| `400` | `invalid_parameter` | Request validation failed | Invalid timestamp, limit > 1000 |
| `401` | `invalid_token` | Authentication failed | Missing or invalid `Authorization` header |
| `403` | `insufficient_permissions` | API key lacks required permissions | Token not provisioned for Live Data |
| `404` | `not_found` | Resource not found | Unknown `market_uid` |
| `422` | `invalid_cursor` | Cursor malformed or expired | Cursor from different query or >24h old |
| `429` | `rate_limit_exceeded` | Too many requests | Exceeded 100 req/min key limit |
| `500` | `internal_error` | Server error | Retry with backoff |
| `503` | `service_unavailable` | Temporary service issue | Retry with exponential backoff |

## Best practices

1. **Poll judiciously:** Fetch recent trades on an interval (e.g., 5–10s) or when streaming indicates fills.
2. **Derive metrics:** Combine trade history with [Top-of-Book Quotes](/pages/live-data/quotes) to compute per-venue slippage and realized spread.
3. **Store cursors:** Persist each venue's `next_cursor` to resume pagination without reprocessing old data.
4. **Backfill with filters:** Use `start`/`end` to pull historical windows for analytics or compliance.
