# PnL
<Callout type="warning" title="Experimental">
This endpoint group is experimental and may change without notice. Access is limited; contact support if you need early integration.
</Callout>

> **GET `/v1/portfolio/pnl`** — Retrieve realized, unrealized, and fee PnL over time with per-venue breakdowns.

- **Base URL:** `https://api.predexon.com/v1`
- **Auth:** `Authorization: Bearer <token>`
- **Time:** All timestamps are **UTC ISO-8601**
- **Rate Limits:** 20 requests/minute per API key. `429` responses include `Retry-After` (seconds).

## Endpoint

```http
GET /v1/portfolio/pnl
```

## Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `granularity` | enum | `day` \| `week` \| `month` \| `year` \| `all`. Default `day`. |
| `start` | string | Inclusive start date/time (`ISO-8601`). Optional. |
| `end` | string | Exclusive end date/time (`ISO-8601`). Optional. |

## Response schema

```json
{
  "granularity": "day",
  "start": "2025-05-01",
  "end": "2025-10-06",
  "curve": [
    {
      "ts": "2025-05-01T00:00:00Z",
      "realized": "22.40",
      "unrealized": "8.10",
      "fees": "-0.40",
      "pnl_to_date": "30.10"
    }
  ],
  "breakdown": {
    "by_venue": {
      "POLYMARKET": "210.7",
      "KALSHI": "95.3"
    }
  }
}
```

## Field reference

| Field | Type | Description |
|-------|------|-------------|
| `granularity` | string | Interval used for aggregation. |
| `start` | string | Start of requested window (inclusive). |
| `end` | string | End of requested window (exclusive). |
| `curve` | array | Time-series points ordered oldest → newest. |
| `curve[].ts` | string (ISO-8601) | Timestamp for the start of the bucket. |
| `curve[].realized` | string | Realized PnL in the bucket. |
| `curve[].unrealized` | string | Unrealized PnL change in the bucket. |
| `curve[].fees` | string | Fees (negative numbers reduce PnL). |
| `curve[].pnl_to_date` | string | Cumulative PnL up to this bucket. |
| `breakdown` | object | Aggregate PnL breakdowns. |
| `breakdown.by_venue` | object | Total PnL by venue. |

<Info>
**Understanding the curve**

- `pnl_to_date` equals the running sum of realized + unrealized + fees.
- When `granularity=all`, the `curve` contains a single cumulative point.
- Unrealized PnL reflects mark-to-market changes on open positions during the bucket.
</Info>

## Examples

<CodeGroup>

```bash cURL
curl -s "https://api.predexon.com/v1/portfolio/pnl?granularity=day&start=2025-06-01&end=2025-09-30" \
  -H "Authorization: Bearer $PREDX_TOKEN"
```

```javascript JavaScript
const base = "https://api.predexon.com/v1";

async function getPnl(params = {}) {
  const url = new URL(`${base}/portfolio/pnl`);
  Object.entries(params).forEach(([key, value]) => url.searchParams.set(key, value));

  const res = await fetch(url, { headers: { Authorization: `Bearer ${process.env.PREDX_TOKEN}` } });
  if (!res.ok) throw new Error(`${res.status} ${await res.text()}`);
  return res.json();
}
```

```python Python
import os
import requests

def get_pnl(**params) -> dict:
  res = requests.get(
      "https://api.predexon.com/v1/portfolio/pnl",
      headers={"Authorization": f"Bearer {os.environ['PREDX_TOKEN']}"},
      params=params,
      timeout=30,
  )
  res.raise_for_status()
  return res.json()

print(get_pnl(granularity="month"))
```

</CodeGroup>

## Errors

All errors return standard HTTP status codes with JSON error responses:

```json
{
  "error": {
    "type": "invalid_parameter",
    "message": "granularity must be one of: day, week, month, year, all",
    "param": "granularity"
  }
}
```

| HTTP Status | Error Type | Description | Common Causes |
|-------------|------------|-------------|---------------|
| `400` | `invalid_parameter` | Request validation failed | Unsupported granularity, `start > end` |
| `401` | `invalid_token` | Authentication failed | Missing or invalid `Authorization` header |
| `403` | `insufficient_permissions` | User has not linked required accounts | Link accounts before requesting PnL |
| `429` | `rate_limit_exceeded` | Too many requests | Exceeded 20 req/min |
| `500` | `internal_error` | Server error | Retry with backoff |
| `503` | `service_unavailable` | Temporary service issue | Retry with exponential backoff |

## Best practices

1. **Align with positions:** Pair PnL curves with [Positions](/pages/portfolio/positions) to explain unrealized swings.
2. **Compare venues:** Use `breakdown.by_venue` to attribute performance and fees per venue. Normalize fees to evaluate venue drag.
3. **Cache curve data:** For dashboards, cache responses per granularity window to avoid heavy recomputation.
4. **Use incremental windows:** Request daily or weekly windows for rolling analytics rather than large all-time pulls.
