# Positions
<Callout type="warning" title="Experimental">
This endpoint group is experimental and may change without notice. Access is limited; contact support if you need early integration.
</Callout>

> **GET `/v1/portfolio/positions`** — Retrieve unified outcome-level holdings across linked venues with live mark prices and margin requirements.

- **Base URL:** `https://api.predexon.com/v1`
- **Auth:** `Authorization: Bearer <token>`
- **Time:** All timestamps are **UTC ISO-8601**
- **Rate Limits:** 40 requests/minute per API key. `429` responses include `Retry-After` (seconds).

## Endpoint

```http
GET /v1/portfolio/positions
```

## Query parameters

All parameters are optional.

| Parameter | Type | Description |
|-----------|------|-------------|
| `venue` | enum | Filter by venue (`POLYMARKET`, `KALSHI`). |
| `market_uid` | string | Filter by canonical market identifier. |

## Response schema

```json
{
  "positions": [
    {
      "market_uid": "mkt_evt_atp_2025_07_14_WIMBLEDON_final_winner_djokovic",
      "venue": "POLYMARKET",
      "outcome": "YES",
      "qty": "320.000000",
      "avg_cost": "0.4410",
      "mark_price": "0.4620",
      "market_value": "147.84",
      "im_margin": "32.00",
      "mm_margin": "25.00"
    }
  ],
  "asof": "2025-10-06T19:21:02Z"
}
```

## Field reference

| Field | Type | Description |
|-------|------|-------------|
| `positions` | array | Outcome-level positions across venues. |
| `positions[].market_uid` | string | Canonical market identifier. |
| `positions[].venue` | string | Venue code (`POLYMARKET`, `KALSHI`). |
| `positions[].outcome` | string | Outcome code held (e.g., `YES`, `NO`, contract ticker). |
| `positions[].qty` | string | Position quantity (shares/contracts). |
| `positions[].avg_cost` | string | Volume-weighted average entry price. |
| `positions[].mark_price` | string | Current mark price used for valuation. |
| `positions[].market_value` | string | Current market value (`qty × mark_price`). |
| `positions[].im_margin` | string | Initial margin requirement, if applicable. |
| `positions[].mm_margin` | string | Maintenance margin requirement, if applicable. |
| `asof` | string (ISO-8601) | Snapshot timestamp. |

<Info>
**Unrealized PnL**

Compute unrealized PnL per position as `(mark_price - avg_cost) × qty`. Combine positions by `market_uid` to derive total market exposure.
</Info>

## Examples

<CodeGroup>

```bash cURL
curl -s "https://api.predexon.com/v1/portfolio/positions?venue=POLYMARKET" \
  -H "Authorization: Bearer $PREDX_TOKEN"
```

```javascript JavaScript
const base = "https://api.predexon.com/v1";

async function listPositions(params = {}) {
  const url = new URL(`${base}/portfolio/positions`);
  Object.entries(params).forEach(([key, value]) => url.searchParams.set(key, value));

  const res = await fetch(url, { headers: { Authorization: `Bearer ${process.env.PREDX_TOKEN}` } });
  if (!res.ok) throw new Error(`${res.status} ${await res.text()}`);
  return res.json();
}
```

```python Python
import os
import requests

def list_positions(**params) -> dict:
  res = requests.get(
      "https://api.predexon.com/v1/portfolio/positions",
      headers={"Authorization": f"Bearer {os.environ['PREDX_TOKEN']}"},
      params=params,
      timeout=30,
  )
  res.raise_for_status()
  return res.json()

print(list_positions(venue="POLYMARKET"))
```

## Errors

All errors return standard HTTP status codes with JSON error responses:

```json
{
  "error": {
    "type": "insufficient_permissions",
    "message": "link an account before requesting positions"
  }
}
```

| HTTP Status | Error Type | Description | Common Causes |
|-------------|------------|-------------|---------------|
| `401` | `invalid_token` | Authentication failed | Missing or invalid `Authorization` header |
| `403` | `insufficient_permissions` | User has not linked required accounts | Call [Link Accounts](/pages/portfolio/link-accounts) first |
| `404` | `not_found` | No positions found | No open positions or filters exclude all data |
| `429` | `rate_limit_exceeded` | Too many requests | Exceeded 40 req/min |
| `500` | `internal_error` | Server error | Retry with backoff |
| `503` | `service_unavailable` | Temporary service issue | Retry with exponential backoff |

## Best practices

1. **Reconcile with balances:** Use collateral (`balances[].collateral`) alongside `im_margin`/`mm_margin` to monitor margin usage.
2. **Group by market:** Aggregate positions by `market_uid` to compute net exposure per market and translate outcomes to plain language via [Market Detail](/pages/discovery/market-detail).
3. **Refresh before trading:** Pull positions immediately after trades or resolutions, or subscribe to [Stream Portfolio](/pages/portfolio/stream) for deltas.
4. **Monitor mark drift:** Large differences between `avg_cost` and `mark_price` indicate unrealized PnL swings; pair with [PnL](/pages/portfolio/pnl) for historical context.
