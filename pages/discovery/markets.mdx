# List Markets

```
GET /v1/markets
```

Enumerate normalized markets across venues. Use this endpoint to discover markets and hop to native IDs when needed.

## Query Parameters

All parameters are optional.

| Parameter | Type | Description |
|-----------|------|-------------|
| `event_uid` | string | Filter by canonical event ID |
| `type` | enum | One of: `binary`, `multi`, `numeric` |
| `trading_status` | enum | One of: `open`, `halted`, `closed` |
| `venue` | enum | One of: `POLYMARKET`, `KALSHI` |
| `polymarket_slug` | string | Match Polymarket slug |
| `polymarket_condition_id` | string | Match Polymarket condition ID |
| `kalshi_event_ticker` | string | Match Kalshi event ticker |
| `limit` | integer | Results per page (default: 50, max: 200) |
| `cursor` | string | Opaque cursor from previous `next_cursor` |

## Example Request

Get all open binary markets for an event:

```bash
curl -s -H "Authorization: Bearer $TOKEN" \
  "https://api.predexa.com/v1/markets?event_uid=evt_mlb_2025_07_12_NYY_BOS&type=binary&trading_status=open"
```

## Notes

- **Lightweight response**: This endpoint returns essential market data. For detailed information (rules, fees, resolution details), use [Market Detail](/pages/discovery/market-detail).
- **Outcomes**: Each outcome has a `code` (e.g., `YES`, `NO`) and may include a `token_id` for venue-specific token references.
- **venue_ids**: Provides native identifiers for hopping to venue-specific systems.
- **tick_size**: Minimum price increment as a string (e.g., `"0.0010"`).
