# Quickstart

Spin up key REST calls in minutes. This guide shows how to resolve venue selectors, list canonical markets, and pull live pricing data using consistent canonical IDs.

## Prerequisites

- Predexon API key (contact support@predexon.com)
- Base REST URL: `https://api.predexon.com/v1`
- WebSocket URL: `wss://stream.predexon.com/v1`
- Environment with curl, Node.js 18+, or Python 3.10+

We’ll use Polymarket and Kalshi selectors for the July 12, 2025 Yankees vs Red Sox game as a concrete example.

## Step 1: Resolve venue selectors to canonical IDs

Use the Discovery Resolve endpoint to convert venue-native selectors into canonical IDs.

<CodeGroup>

```bash cURL
curl -s -X POST "https://api.predexon.com/v1/resolve" \
  -H "Authorization: Bearer $PREDX_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "selectors": [
      {"venue": "polymarket", "market_slug": "will-yankees-win"},
      {"venue": "kalshi", "market_ticker": "KXMLBNYY-25JUL12"}
    ]
  }'
```

```javascript JavaScript (resolve)
const base = "https://api.predexon.com/v1";
const headers = {
  Authorization: `Bearer ${process.env.PREDX_TOKEN}`,
  "Content-Type": "application/json",
};

async function resolveSelectors() {
  const body = {
    selectors: [
      { venue: "polymarket", market_slug: "will-yankees-win" },
      { venue: "kalshi", market_ticker: "KXMLBNYY-25JUL12" },
    ],
  };

  const res = await fetch(`${base}/resolve`, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
  });
  if (!res.ok) throw new Error(`${res.status} ${await res.text()}`);
  return res.json();
}
```

```python Python (resolve)
import os
import requests

BASE = "https://api.predexon.com/v1"
HEADERS = {
    "Authorization": f"Bearer {os.environ['PREDX_TOKEN']}",
    "Content-Type": "application/json",
}

payload = {
    "selectors": [
        {"venue": "polymarket", "market_slug": "will-yankees-win"},
        {"venue": "kalshi", "market_ticker": "KXMLBNYY-25JUL12"},
    ]
}

res = requests.post(f"{BASE}/resolve", headers=HEADERS, json=payload, timeout=30)
res.raise_for_status()
print(res.json())
```

</CodeGroup>

Response (abridged):

```json
{
  "results": [
    {
      "market_uid": "mkt_evt_mlb_2025_07_12_NYY_BOS_winner_nyy",
      "event_uid": "evt_mlb_2025_07_12_NYY_BOS"
    }
  ]
}
```

Store `market_uid` and `event_uid`—you’ll reuse them in later steps. See [Resolve Selectors](/pages/discovery/resolve) for full schema and batching tips.

## Step 2: List canonical markets for the event

Use the `event_uid` from Step 1 to pull normalized markets.

<CodeGroup>

```bash cURL
curl -s "https://api.predexon.com/v1/markets?event_uid=evt_mlb_2025_07_12_NYY_BOS" \
  -H "Authorization: Bearer $PREDX_TOKEN"
```

```javascript JavaScript (markets)
const base = "https://api.predexon.com/v1";

async function listMarkets(eventUid) {
  const url = new URL(`${base}/markets`);
  url.searchParams.set("event_uid", eventUid);

  const res = await fetch(url, {
    headers: { Authorization: `Bearer ${process.env.PREDX_TOKEN}` },
  });
  if (!res.ok) throw new Error(`${res.status} ${await res.text()}`);
  return res.json();
}
```

```python Python (markets)
import os
import requests

def list_markets(event_uid: str) -> dict:
  res = requests.get(
      "https://api.predexon.com/v1/markets",
      headers={"Authorization": f"Bearer {os.environ['PREDX_TOKEN']}"},
      params={"event_uid": event_uid},
      timeout=30,
  )
  res.raise_for_status()
  return res.json()

print(list_markets("evt_mlb_2025_07_12_NYY_BOS"))
```

</CodeGroup>

## Step 6 (Optional): Live quotes over WebSocket

```bash
wscat -c "wss://stream.predexon.com/v1/quotes" \
  -H "Authorization: Bearer $PREDX_TOKEN"
```

After connecting, subscribe to the market:

```json
{"action":"subscribe","market_uids":["mkt_evt_mlb_2025_07_12_NYY_BOS_winner_nyy"]}
```

See [Live Data Streams](/pages/live-data/stream-quotes) for full message formats and authentication details.

## Troubleshooting

- **401 / 403** – Confirm your API key and token scope
- **429** – Respect per-key rate limits (100 req/min). Use exponential backoff
- **Pagination** – Follow `next_cursor` links on list responses
- **Field reference** – Consult the Discovery and Live Data docs linked above
